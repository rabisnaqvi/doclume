# Doclume marketing design

## Goal
Get Doclume its first users in 30 days with low-key marketing. Treat the web app and VS Code extension as one product story.

## Constraints
- No existing social presence.
- Low comfort with marketing.
- 1-2 hours/week, with AI help.
- Target audience: developers.
- No paid ads for now.

## Chosen approach
**Assets + community bursts.**

Build owned assets first, then do a small number of targeted community posts. This fits the time budget and avoids depending on a personal brand.

## Positioning
- Doclume = a markdown reader that respects typography, not just a markdown renderer.
- Core hook: **Five themes. Zero Comic Sans.**
- Support line: **Markdown preview that respects typography.**
- Primary audience: developers writing README files, changelogs, specs, and docs.
- Proof points: 5 themes, auto mode, serif/sans options, high contrast, browser + VS Code entry points.

## Channels + funnel
### Owned assets
- Root README
- VS Code extension README
- Marketplace listing
- Web app landing page

### Community bursts
- 1-2 targeted posts in developer spaces
- Reuse the same asset with a different angle if the first post does not land

### Funnel
post/thread -> README or landing page -> install/open -> first read -> feedback

## 30-day plan
### Week 1
- Tighten positioning copy
- Make screenshots/demos easy to understand fast
- Ensure install/open CTA is obvious

### Week 2
- Publish first post or thread
- Share in 1-2 relevant communities
- Watch clicks and comments, not vanity likes

### Week 3
- Collect feedback
- Fix friction in copy, screenshots, or onboarding
- If one angle wins, sharpen it

### Week 4
- Run a second push using the best-performing angle
- Decide which channel deserves the next month

## Goals
### Floor
- 5 first users / installs or equivalent web opens
- 3 real feedback conversations

### Stretch
- 10-20 installs
- 5 feedback conversations
- 1 repeatable channel that can be reused

## Weekly cadence
- 30 min: improve one asset
- 30 min: one post or one community comment
- 30 min: reply to feedback
- optional 15 min: note what worked and what did not

## Risks + fallback
- If posts flop, reuse the same asset with a different angle.
- If installs lag, push the web app first and extension second.
- If visuals win, spend more time on screenshots/GIFs and less on copy.
- If nothing bites, reduce to README + one post + one feedback ask per week.

## Out of scope for now
- Paid ads
- Influencer buys
- Building a large social presence first
